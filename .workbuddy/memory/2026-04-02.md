## 彻底修复卡片展开布局问题（15:21）

### 问题根因
之前的方案用 `grid-column: 1 / -1` 让展开的卡片占满一行，但这会把后面的卡片挤到下一行，导致排列混乱（第3、4个卡片被挤到下方）。

### 解决方案：面板分离架构
彻底重构 HTML/CSS/JS：
1. **HTML结构改变**：把三个展开内容（价格表、预约提示、必去推荐）从各自的 `.feature-card` 里移出，放到卡片组 `.features` 之外的独立容器 `.feature-panel-wrap` 里，每个面板有独立 ID（panelPrice/panelBooking/panelRecommend）
2. **CSS改变**：`.features` 保持固定4列网格，永远只放4个卡片；展开面板用 `.feature-panel { display:none }` / `.feature-panel.active { display:block }` 控制
3. **JS改变**：点击卡片时操作对应面板的 `active` class，而不是操作卡片自身的 `expanded` class；卡片只加 `active-card` class（视觉下沉+高亮效果）

### 效果
- 四个卡片始终固定在同一排，互不干扰
- 点击卡片后，内容在卡片组下方完整展开，无多余空白
- 点击同一卡片再次收起，切换卡片时平滑过渡
- 点击外部区域自动关闭
