## 彻底修复地图黑屏问题（15:30）

### 问题根因
昨晚的优化脚本（optimize_routes_v2.py）在写入推荐路线数据时，将多行文本内容（day/schedule 字段）
用普通单引号字符串保存，但字符串内包含了换行符（\n）。
这在 JavaScript 中是严格的语法错误：普通字符串不能跨行，只有反引号模板字符串才支持。
结果：整个 <script> 代码块解析失败，Leaflet 无法初始化，地图黑屏。

### 修复方案
用 Python 脚本（fix_strings.py）自动检测并修复所有多行单引号字符串，
将开头和结尾的单引号改为反引号（），共修复 18 处。

### 验证
- check_js.py 重新扫描：No unclosed string errors detected
- 本地预览：地图正常显示
- Git commit: a0967e8
- 已推送到 GitHub Pages，自动更新

### 经验教训
以后用 Python 写多行字符串到 JS 时，必须用反引号模板字符串，不能用普通引号。
